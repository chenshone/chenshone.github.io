    从暴力递归到动态规划 方法论

    CHENSHONE 2020/11/4 算法思维

    # 暴力递归存在的问题

    暴力递归在递归过程中，存在大量重复计算，浪费了很多时间。 所以，可以采用一个备忘录，将之前计算的结果记录起来，下次要用的时候，直接取出即可，这样就节约了时间

    # 什么样的暴力递归可以优化

    1. 有重复调用同一个子问题的解，这种递归可以优化
    2. 如果每一个子问题都是不同的解，无法优化也不用优化

    # 暴力递归和动态规划的关系

    1. 某一个暴力递归，有解的重复调用，就可以把这个暴力递归优化成动态规划
    2. 任何动态规划问题，都一定对应着某一个有解的重复调用的暴力递归
    3. 但不是所有的暴力递归，都一定对应着动态规划

    # 如何找出题目的dp做法

    1. 设计暴力递归：重要原则+4种常见尝试模型
    2. 分析有没有重复解
    3. 用记忆化搜索 -> 用严格表结构实现动态规划
    4. 看看能否继续优化(去掉循环、dp降维等)

    # 设计暴力递归过程的原则

    1. 每一个可变参数的类型，一定不要比int类型更加复杂 -> 每个状态要方便构造成dp表
    2. 原则1）可以违反，让类型突破到一维线性结构，那必须是单一可变参数 -> 也只能是记忆化搜索的方法
    3. 如果发现原则1）被违反，但不违反原则2），只需要做到记忆化搜索即可
    4. 可变参数的个数，能少则少

    # 暴力递归到动态规划

    1. 你已经有了一个不违反原则的暴力递归，而且的确存在解的重复调用
    2. 找到哪些参数的变化会影响返回值，对每一个列出变化范围
    3. 参数间的所有的组合数量，意味着表大小 -> 暴力递归函数的形参中有几个可变参数，那么dp就设几个维度的表
    4. 记忆化搜索的方法就是傻缓存，非常容易得到 -> memo可以是数组(简单)，也可以是hashmap(复杂，key是字符串啥的)
    5. 规定好严格表的大小，分析位置的依赖顺序，然后从基础填写到最终解
      1. 暴力递归的base case，也就是dp的base case
      2. 暴力递归的决策过程，也就是dp的动态转移方程
    6. 对于有枚举行为的决策过程，进一步优化
      1. 空间压缩
      2. 状态化简
      3. 四边形不等式
      4. 等等

    # 暴力尝试例题 （与dp无关，因为用到回溯）

    # N皇后

    N皇后问题是指在N*N的棋盘上要摆N个皇后 要求任何两个皇后不同行、不同列，也不在同一条斜线上 给定一个整数n，返回n皇后的摆法有多少种

    # 解法一

    public static int num1(int n) {
    if (n < 1) {
        return 0;
    }
    int[] record = new int[n]; // record[i] -> i行的皇后放在了record[i]位置上
    return process1(0, record, n);
}

/**
  * @param i      目前来到第i行
  * @param record 皇后摆放的位置
  * @param n      一共有几行
  * @return 返回摆完所有皇后之后，合法的摆法有多少种
  */
private static int process1(int i, int[] record, int n) {
    if (i == n) {
        return 1; // 得到一种摆法
    }
    int res = 0;
    for (int j = 0; j < n; j++) { // 当前在第i行，从左往右尝试所有列
        if (isValid(record, i, j)) {
            // 每行只放一个，规避掉不能同行的限制
            record[i] = j;
            res += process1(i + 1, record, n);
        }
    }
    return res;
}

/**
  * 在棋盘上[i, j]位置是否能摆放皇后
  *
  * @param record 摆放皇后的位置
  * @param i      行
  * @param j      列
  * @return 当前位置是否能摆放
  */
private static boolean isValid(int[] record, int i, int j) {
    for (int k = 0; k < i; k++) {
        // 不能同列
        // 不能同斜线 -> 斜率不能为(+/-)1
        if (j == record[k] || Math.abs(record[k] - j) == Math.abs(i - k)) {
            return false;
        }
    }
    return true;
}

    # 解法二 位运算优化

    // 因为用到位运算 所以不能超过32皇后问题
public static int num2(int n) {
    if (n < 1 || n > 32) {
        return 0;
    }
    // 如果是8皇后问题， limit最右8个1，其余都是0
    int limit = n == 32 ? -1 : (1 << n) - 1;
    return process2(limit, 0, 0, 0);
}

/**
  * @param limit       划定问题的规模 -> 固定变量
  * @param colLim      列的限制，1的位置表示已经放了皇后，不能再放皇后
  * @param leftDiaLim  左斜线的限制
  * @param rightDiaLim 右斜线的限制
  * @return 返回摆完所有皇后之后，合法的摆法有多少种
  */
public static int process2(int limit, int colLim, int leftDiaLim, int rightDiaLim) {
    if (colLim == limit) { // 已经放满了皇后
        return 1;
    }

    // 所有候选皇后的位置，都在pos上
    // colLim | leftDiaLim | rightDiaLim 当前层不能放皇后的位置
    // ~(colLim | leftDiaLim | rightDiaLim) 左侧为1的为干扰，右侧为1的就是可以摆放皇后的位置
    // 与limit 进行与运算，正好将左侧为1的重置为0，不再干扰
    int pos = limit & (~(colLim | leftDiaLim | rightDiaLim));
    int mostRightOne = 0;
    int res = 0;
    while (pos != 0) {
        mostRightOne = pos & (~pos + 1); // 提取最右侧的1，其余位都是0
        pos = pos - mostRightOne;
        res += process2(limit,
                colLim | mostRightOne,
                // 本来的左斜线限制或上本层放皇后的位置就是当前层的左限制，往左移一位，就是下一层的左限制
                (leftDiaLim | mostRightOne) << 1,
                // 本来的右斜线限制或上本层放皇后的位置就是当前层的右限制，无符号右移一位，就是下一层的右限制
                (rightDiaLim | mostRightOne) >>> 1);
    }
    return res;
}
