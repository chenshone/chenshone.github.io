    数组累加和问题

    CHENSHONE 2020/12/1 算法思维

    # 题目一

    给定一个全是正整数的数组arr，一个数sum，问累加和等于sum的子数组最长有多长

    # 思路

    正数的累加和有单调性，所以可以用双指针、滑动窗口等方法解决

    本题可以用滑动窗口来解决

    设置一个滑动窗口，其左边界为l，右边界为r，窗口和为windowSum

    1. windowSum < sum, r++
    2. windowSum > sum, l++
    3. windowSum == sum, 更新ans、l++

    # 代码

    public static int getMaxLength(int[] arr, int K) {
    if (arr == null || arr.length == 0 || K <= 0) {
        return 0;
    }
    // [L, R]
    int L = 0;
    int R = 0;
    int sum = arr[0];
    int len = 0;
    while (R < arr.length) {
        if (sum == K) {
            len = Math.max(len, R - L + 1);
            sum -= arr[L++];
        } else if (sum < K) {
            R++;
            if (R == arr.length) {
                break;
            }
            sum += arr[R];
        } else {
            sum -= arr[L++];
        }
    }
    return len;
}

    # 题目二

    给定一个全是整数（有正、有负、有0）的数组arr，一个数sum，问累加和等于sum的子数组最长有多长

    # 思路

    单调性没有了，换一种思路，求以某个下标为结尾的子数组，判断该子数组是否符合条件，只要求得所有情况，那么答案必在其中

    那么这个子数组怎么快速求呢，用前缀和数组，两个前缀和数组相减就能得到某个区间数组，

    而怎么求得某个sum的最长呢，

    假设：[0, 10]是sum是1000，且[0, 3]的sum是200，且是第一次出现200，那么[4, 10]就是[0, 10]区间上长度最长的sum为800的子数组

    # 代码

    public static int maxLength(int[] arr, int k) {
    if (arr == null || arr.length == 0) {
        return 0;
    }
    // key 前缀和
    // value 最早出现的位置（结尾位置）
    HashMap<Integer, Integer> map = new HashMap<>();
    map.put(0, -1);
    int len = 0;
    int sum = 0;
    for (int i = 0; i < arr.length; i++) {
        sum += arr[i];
        if (map.containsKey(sum - k)) {
            len = Math.max(i - map.get(sum - k), len);
        }
        if (!map.containsKey(sum)) {
            map.put(sum, i);
        }
    }
    return len;
}

    # 拓展

    现有一个全是整数（有正、有负、有0）的数组arr，求子数组中1的数量和2的数量相等的子数组，问最长的达标子数组的长度

    遍历数组，将无关元素置为0，1还是1，2变成-1，那么问题就变成求累加和为0的子数组中，最长为多长

    # 题目三

    给定一个全是整数（有正、有负、有0）的数组arr，一个数sum，问累加和 小于等于 sum的子数组最长有多长

    # 思路

    假设当前喂鱼数组位置为i，记录以i开头的所有子数组中，累加和最小的范围

    设置minSum[] 和 minSumEnd[]

    minSum[i]表示i开头的所有子数组中，累加和最小的

    minSumEnd[i]表示i开头的所有子数组中，累加和最小的那个右边界

    得到这两个数组后，

    假设现在是找以0开头的，[0, 7]、[8, 13]、[14, 29]的区间和（每个都是以某个开头的最小累加和的区间）加起来都比sum小，而加上[30, ..]就比sum大了，

    那么可以得到[0, 29]是符合要求的，

    接下来，可以得到[1, 29] 然后再加上[30, ..]看看是不是符合条件

    这里之所以直接得到[1, 29]再往后比较判断，是因为，假设1开头的得到的符合条件的比0开头的符合条件的短，那么这个结果也不是想要的（因为要求最长），所以可以直接从[1, 29]开始往后，这叫舍弃可能性

    # 代码

    public static int maxLength(int[] arr, int k) {
    if (arr == null || arr.length == 0) {
        return 0;
    }
    int[] minSums = new int[arr.length];
    int[] minSumEnds = new int[arr.length];
    minSums[arr.length - 1] = arr[arr.length - 1];
    minSumEnds[arr.length - 1] = arr.length - 1;
    for (int i = arr.length - 2; i >= 0; i--) {
        if (minSums[i + 1] <= 0) {
            minSums[i] = arr[i] + minSums[i + 1];
            minSumEnds[i] = minSumEnds[i + 1];
        } else {
            minSums[i] = arr[i];
            minSumEnds[i] = i;
        }
    }
    int end = 0;
    int sum = 0;
    int res = 0;
    // i是窗口的最左位置，end扩出来的最右有效块的最后一个位置的再下一个位置
    // end也是下一块的开始位置
    // 窗口 [i, end)
    for (int i = 0; i < arr.length; i++) {
        // 扩到不能扩位置
        while (end < arr.length && sum + minSums[end] <= k) {
            sum += minSums[end];
            end = minSumEnds[end] + 1;
        }
        res = Math.max(res, end - i);
        if (end > i) { // 窗口内还有数
            sum -= arr[i];
        } else {
            end = i + 1;
        }
    }
    return res;
}
