    打表技巧

    CHENSHONE 2020/11/25 算法思维

    适用情况

    1. 问题如果返回值的情况(种类)不太多，可以用hardcode方式列出，作为程序的一部分
    2. 一个大问题解决时底层频繁使用规模不大的小问题的解，如果小问题的返回值满足条件1，可以把小问题的解列成一张表，作为程序的一部分
    3. 打表找规律

    # 打表找规律

    1. 对于一个题，输入参数类型很简单，并且只有一个实际参数
    2. 要求的返回值类型也简单，并且只有一个
    3. 用暴力方法，把输入参数对应的返回值，打印出来看看，进而优化code

    # 题目一

    小明去买苹果，商店只提供两种类型的塑料袋，每种类型都有任意数量

    1. 能装下6个苹果的袋子
    2. 能装下8个苹果的袋子 小明可以自由使用两种袋子来装苹果，但是小明有强迫症，他要求自己使用的袋子数量必须最少，且使用的每个袋子必须装满 给定一个正整数N，返回至少使用多少个袋子。如果N无法让使用的每个袋子必须装满，返回-1

    # 暴力解

    public static int minBags(int apple) {
    if (apple < 0) {
        return -1;
    }
    int bag6 = -1;
    int bag8 = apple / 8;
    int rest = apple - 8 * bag8;
    while (bag8 >= 0 && rest < 24) { // 24 是8和6的最小公倍数，rest大于这个值就没必要算了，必然没结果的
        int restUse6 = minBagBase6(rest);
        if (restUse6 != -1) {
            bag6 = restUse6;
            break;
        }
        rest = apple - 8 * (--bag8);
    }
    return bag6 == -1 ? -1 : bag6 + bag8;
}

private static int minBagBase6(int rest) {
    return rest % 6 == 0 ? (rest / 6) : -1;
}

    # 找到规律后的解

    public static int minBagAwesome(int apple) {
    if ((apple & 1) != 0) {
        return -1;
    }
    if (apple < 18) {
        return apple == 0 ? 0 : (apple == 6 || apple == 8) ? 1 : (apple == 12 || apple == 14 || apple == 16) ? 2 :
                -1;
    }

    return (apple - 18) / 8 + 3;
}

    # 题目二

    给定一个正整数M，表示有M份青草统一堆放在仓库里 有一只牛和一只羊，牛先吃，羊后吃，他俩轮流吃草 不管是牛还是羊，每一轮能吃的草量必须是4^n（n∈N) 谁最先把草吃完，谁获胜 假设牛和羊都绝顶聪明，都想赢，都会做出理性的决定 根据唯一的参数M，返回谁会赢

    # 暴力解

    // n份青草放在一堆
// 先手后手都绝顶聪明
// return "先手" / "后手"
public static String winner1(int n) {
    // 0  1  2  3  4
    // 后 先 后  先 先
    if (n < 5) { // base case
        return (n == 0 || n == 2) ? "后手" : "先手";
    }
    // n >= 5 时
    int base = 1; // 当前先手决定吃的草数
    // 当前是先手在选
    while (base <= n) {
        // 当前一个n份草，先手吃掉的是base份，n - base 是给后手的
        // 母过程 先手 在子过程里是 后手
        if (winner1(n - base).equals("后手")) {
            return "先手";
        }
        if (base > n / 4) { // 放置 base*4之后溢出
            break;
        }
        base *= 4;
    }
    return "后手";
}

    # 题目三

    定义一种数：可以表示成若干（数量>1）连续正数和的数 比如： 5 = 2 + 3 12 = 3 + 4 + 5 1不是，因为要求数量大于1个 2 = 1 + 1 ，也不是，因为不是连续正数和 给定一个参数N，返回是不是可以表示成若干连续正数和的数

    # 暴力解

    public static boolean isMSum1(int num) {
    for (int i = 1; i <= num; i++) {
        int sum = i;
        for (int j = i + 1; j <= num; j++) {
            if (sum + j > num) {
                break;
            }
            if (sum + j == num) {
                return true;
            }
            sum += j;
        }
    }
    return false;
}

    可以得到如下规律

    2的某次幂为false，其余全是true

    # 找到规律后的解

    // (num & (num - 1)) == 0 就意味着 num是2的某次方
// 因为如果num是2的某次方，那么对应的二进制，只有一个1
// 那么减一的话，就会把这个1消掉，然后后面的所有位都变为1
// 这样&的话 就会是0
// 如果不止有1个位是1，那么减一的话，就只会把最低位的那个1消掉，后面变成0
// 不会影响高位的1，那么 & 的结果就不会是0
public static boolean isMSum2(int num) {
    if (num < 3) {
        return false;
    }
    return (num & (num - 1)) != 0;
}
