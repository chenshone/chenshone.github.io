    排序(三) 计数、基数、排序总结

    CHENSHONE 2020/10/27 算法思维sort

    不作任何说明的话，排序都按升序来。

    # 计数排序 稳定

    计数排序的前提是直到数据中的最大值（假设没负数），适用于小规模。

    # 思路

    比方说，一个数组arr中的最大值为n，开辟一块n+1的数组tmp，遍历待排序数组，每遍历一个数arr[i]，就将tmp[arr[i]]++,遍历完，遍历tmp数组，如果当前值>0，就将原数组arr从前往后覆盖等量的tmp当前的下标。

    # 代码

    public class CountSort {
    public static void countSort(int[] arr) {
        if (arr == null || arr.length < 2) {
            return;
        }

        int max = Integer.MIN_VALUE;
        for (int i = 0; i < arr.length; i++) {
            max = Math.max(max, arr[i]);
        }
        int[] bucket = new int[max + 1];
        for (int i = 0; i < arr.length; i++) {
            bucket[arr[i]]++;
        }
        int i = 0;
        for (int j = 0; j < bucket.length; j++) {
            while ((bucket[j]--) > 0) {
                arr[i++] = j;
            }
        }

    }
}

    # 基数排序 稳定

    # 思路

    准备0 - 9十个队列，找出待排序数组中最大的那个，计算出他是几位数，将其他位数不足的前面补0，从个位数开始，数组从左向右依次按照个位数的数组放入相应的队列中，一轮结束后，按照从小到大的顺序依次弹出每个队列覆盖原数组，然后按十位，百位。。。依次重复操作，直到所有位数都执行一遍，排序完成。

    # 代码

    public class RadixSort {
    //    only fot no-negative value
    public static void radixSort(int[] arr) {
        if (arr == null || arr.length < 2) {
            return;
        }
        radixSort(arr, 0, arr.length - 1, maxbits(arr));
    }

    private static void radixSort(int[] arr, int l, int r, int digit) {
        final int radix = 10;
        int i = 0, j = 0;
//        有多少个数准备多少个辅助空间
        int[] help = new int[r - l + 1];
        for (int d = 1; d <= digit; d++) { // 有多少位就进出几次
            /*
             * 10个空间
             * count[0] 当前位(d位)是0的数字有多少个
             * count[1] 当前位(d位)是0和1的数字有多少个
             * count[2] 当前位(d位)是0、1和2的数字有多少个
             * */
            int[] count = new int[radix]; // count[0-9]
//            求前缀和
            for (i = 1; i < radix; i++) {
                count[i] = count[i] + count[i - 1];
            }
            /*
             * 从右往左遍历，
             * 按照原思路，从左往右依次入队，先进先出，
             * 那么，从右往左遍历的话，对于当前遍历的位数i，
             * 第一个遍历到的，就应该是原本最后一个出队的，也就是原本应该排在当前位数这个值的最后位置的数
             * 而根据之前的count数组求出的前缀和，可以知道当前位数，小于等于它的有几个，
             * 而这几个，就是排在最前面的
             * 而当前遍历到的，是这几个当做最靠右的（最大的），
             * 显然，就能得到当前数应该在的位置
             * */
            for (i = r; i >= l; i--) {
                j = getDigit(arr[i], d);
                help[count[j] - 1] = arr[i];
                count[j]--;
            }
            for (i = l, j = 0; i <= r; i++, j++) {
                arr[i] = help[j];
            }
        }

    }

    private static int getDigit(int x, int d) {
        return ((x / ((int) Math.pow(10, d - 1))) % 10);
    }

    private static int maxbits(int[] arr) {
        int max = Integer.MIN_VALUE;
        for (int i = 0; i < arr.length; i++) {
            max = Math.max(arr[i], max);
        }
        int res = 0;
        while (max != 0) {
            res++;
            max /= 10;
        }
        return res;
    }
}

    # 复杂度

    时间复杂度：O(n*log10n) -> O(n)

    空间复杂度：O(m)

    # 总结

    1. 不基于比较的排序，对样本数据有严格要求，不易改写
    2. 基于比较的排序，时间复杂度的极限是O(N*logN)
    3. 只追求速度选快排，有空间要求选堆排，有稳定性要求选归并

    常见的坑

    1. 归并排序的额外空间复杂度可以变成O(1),"归并排序内部缓存法"，但是将变得不再稳定，不如直接堆排
    2. 原地归并排序是垃圾，会让时间复杂度变成O(N^2)
    3. 快排稳定性改进，"01 stable sort"，但是会对样本数据要求更多
