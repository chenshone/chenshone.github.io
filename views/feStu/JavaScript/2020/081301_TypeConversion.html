    JavaScript中的类型转换（一）

    CHENSHONE 2020/8/13 JavaScript

    # 什么是类型转换

    将值从一种类型转换为另一种类型通常称为类型转换（type casting），这是显式的情况；隐式的情况称为强制类型转换（coercion）。

    JavaScript 中的强制类型转换总是返回标量基本类型值，如字符串、数字和布尔值，不会返回对象和函数。

    在js中，可以将类型转换分为显式/隐式强制类型转换。

    const a = 1
const b = a + '' //  隐式强制类型转换
const c = String(a) // 显式强制类型转换

    如上方代码展示的那样，对于 显式强制类型转换 我们可以很清楚的看出来，数字a被转成了字符串。

    而对于 隐式强制类型转换 来说，就不是那么明显了，它是由于 + 运算符的其中一个操作数是字符串，所以是字符串拼接操作，结果是数字 1 被强制类型转换为相应的字符串 "1"。

    # 抽象值操作

    ES5 规范第 9 节中定义了一些“抽象操作”（即“仅供内部使用的操作”）和转 换规则。

    # ToString

    ToString 负责处理非字符串到字符串的强制类型转换。

    这个方法很常见，就是将非字符串转成字符串，不过有一点需要注意的是：

    1. 对于大数，toString 会采用科学计数法表示；
    2. 对于普通的object，会返回 "[object Object]"
    3. 对于数组，比如 [1, 2, 3], toString 后会变成 "1,2,3"
    4. 当然，你也可以自己重写 toString 方法

    # ToNumber

    对于基本类型的值，处理的很简单：

    转换前 转换后
    true 1
    false 0
    undefined NaN
    null 0

    对于字符串，基本遵循数字常量的相关规则 / 语法(如果字符串内容是纯数字的话，则会处理成功)，处理失败的话，返回 NaN。

    对于对象(包括数组)，会先转成相应的基本类型，如果相应的基本类型是非数字的话，再按照上面的规则转换。

    对于对象(包括数组)，想要转成相应的基本类型，需要先检查该值是否有 valueOf() 方法。如果有并且返回基本类型值，就使用该值进行强制类型转换。如果没有就使用 toString() 的返回值（如果存在）来进行强制类型转换。

    如果 valueOf() 和 toString() 均不返回基本类型值，会产生 TypeError 错误。

    ::: waring 使用 Object.create(null) 创建的对象 [[Prototype]] 属性为 null ，并且没有 valueOf() 和 toString() 方法，因此无法进行强制类型转换。 :::

    # ToBoolean

    这些值可以被强制类型转换为 false：

    可以被强制类型转换的值
    undefined
    null
    false
    +0
    -0
    ""
    NaN

    反过来，其他值就被转为 true 。

    # 假值对象

    对于封装了假值的对象，他们强转后，都是 true。

    const a = new Boolean(false)
const b = new Boolean(0)
const c = new Boolean('')
console.log(Boolean(a && b && c)) // true, 说明a , b, c 都为 true
console.log(Boolean(false && 0 && '')) // false

    # 字符串和数字之间的显式转换

    数字转字符串 解释
    String(a) String函数
    a.toString() toString方法
    字符串转数字 解释
    Number(a) Number函数
    +a 一元运算符 + （建议不要和其他运算符在一起使用）
    - -a 一元运算符 -，和 + 一样，并且它还会反转数字的符号位，所以前面要再加一个 - 号，但是 -- 是自减运算符，所以中间需要有一个空格间隔开（不推荐使用）
    ~~a 取反运算符 ~， ~a相当于 -(a + 1)，在一些方法中，-1 是作为判断的一个临界点(哨兵)，可以使用取反运算符，也可以用来截除数字值的小数部分（相当于Math.trunc）

    # 显式解析数字字符串

    解析数字字符串 解释
    Number(a) Number函数
    parseInt(a) 解析允许字符串中含有非数字字符，解析按从左到右的顺序，如果遇到非数字字符就停止。而转换不允许出现非数字字符，否则会失败并返回 NaN
    parseFloat(a) 解析允许字符串中含有非数字字符，解析按从左到右的顺序，如果遇到非数字字符就停止。而转换不允许出现非数字字符，否则会失败并返回 NaN

    对于 parseInt 函数，有一些很有意思的例子

    parseInt( 1/0, 19 ); // 18 ("I" 来自于 "Infinity"，在以19位基数的表示中，最大的数是 I，即18)
parseInt( 0.000008 ); // 0 ("0" 来自于 "0.000008")
parseInt( 0.0000008 ); // 8 ("8" 来自于 "8e-7")
parseInt( false, 16 ); // 250 ("fa" 来自于 "false")
parseInt( parseInt, 16 ); // 15 ("f" 来自于 "function..")
parseInt( "0x10" ); // 16
parseInt( "103", 2 ); // 2

    # 显式转换为布尔值

    解析数字字符串 解释
    Boolean(a) Boolean函数
    !!a 一元运算符 !

    对于if语句、三元表达式、for中需要判断的地方，如果没有使用 Boolean() 和 ! ，会自动隐式转换。
