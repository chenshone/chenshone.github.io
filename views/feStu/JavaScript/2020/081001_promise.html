    # Promise (opens new window) 是什么

    Promise 是ES6中进行异步编程的新的解决方案。

    # 基本使用

    const promise1 = new Promise((resolve, reject) => {
  setTimeout(() => {
    if (Date.now()) {
      resolve('ok')
    }
    reject('err')
  }, 300)
})

promise1.then((value) => {
  console.log(value)
  // expected output: "ok"
}, (err) => {
  console.log(err)
  // expected output: "err"
})

console.log(promise1)
// expected output: [object Promise]

    # 使用 Promise 的好处

    以前处理异步任务是采用纯回调函数的方式:

    // 成功的回调函数
function successCallback(result) {
  console.log("音频文件创建成功: " + result)
}

// 失败的回调函数
function failureCallback(error) {
  console.log("音频文件创建失败: " + error)
}

createAudioFileAsync(audioSettings, successCallback, failureCallback)

    缺点：

    1. 回调函数必须在启动异步任务前指定
    2. 回调函数嵌套回调函数，会造成令人恶心的 “回调地狱” 问题。不便于阅读与异常处理

    # Promise 解决方案

    # 链式调用

    const promise = doSomething()
// then() 函数会返回一个和原来不同的 **新的Promise** ：
const promise2 = promise.then(successCallback, failureCallback)

    # Catch 的后续链式操作

    then里的参数是可选的，catch(failureCallback) 是 then(null, failureCallback) 的缩略形式。

    有可能会在一个回调失败之后继续使用链式操作，即 使用一个 catch，这对于在链式操作中抛出一个失败之后，再次进行新的操作很有用。

    new Promise((resolve, reject) => {
    console.log('初始化');

    resolve();
})
.then(() => {
    throw new Error('有哪里不对了');

    console.log('执行「这个」”');
})
.catch(() => {
    console.log('执行「那个」');
})
.then(() => {
    console.log('执行「这个」，无论前面发生了什么');
});

    # 错误传递

    doSomething()
.then(result => doSomethingElse(value))
.then(newResult => doThirdThing(newResult))
.then(finalResult => console.log(`Got the final result: ${finalResult}`))
.catch(failureCallback);

    通常，一遇到异常抛出，浏览器就会顺着promise链寻找下一个 onRejected 失败回调函数或者由 .catch() 指定的回调函数。也可以使用ES7中的async/await达到同样的效果。

    try {
  let result = syncDoSomething();
  let newResult = syncDoSomethingElse(result);
  let finalResult = syncDoThirdThing(newResult);
  console.log(`Got the final result: ${finalResult}`);
} catch(error) {
  failureCallback(error);
}
