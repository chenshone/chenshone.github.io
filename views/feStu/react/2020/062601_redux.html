    Redux——状态管理工具

    CHENSHONE 2020/6/26 redux

    # 理解

    # 是什么

    1. redux是一个独立专门用于做状态集中管理的JS库(不是react插件库)
    2. 他可以用在react，angular，vue等项目中，但基本与react配合使用
    3. 作用：集中式管理react应用中多个组件共享的状态

    # redux工作流程

    图片详情 (opens new window)

    # 什么时候需要

    1. 总体原则：能不用就不用，如果不用比较吃力才考虑使用
    2. 某个组件的状态需要共享
    3. 某个状态需要在任何地方都可以拿到
    4. 一个组件需要改变全局状态
    5. 一个组件需要改变另一个组件的状态

    # redux的核心概念

    # action（声明，描述动作的类型以及携带的相关数据）

    Action是把数据从应用传到store的有效载荷。它不负责更改数据，只负责更改行为的声明(即要执行那个动作，执行动作携带的数据)他是store数据的唯一来源。一般来说会通过store.dispatch()将action传到store。

    • 本质: JavaScript 普通对象
    • 约定: action 内必须使用一个字符串类型的 type 字段来表示将要执行的动作。多数情况下，type 会被定义成字符串常量。除了 type 字段外，action 对象的结构完全由你自己决定，一般来说会有一个payload字段。
    • 定义: 创建一个函数, 目的是动态给action设置payload
    function addTodo(payload) {
  return {
    type: ADD_TODO,
    payload
  }
}

    # reducer（描述具体响应动作）

    Reducers指定了应用状态的变化如何响应actions并发送到store的, 记住actions只是描述了有事情发生了这一事实, 并没有描述应用如何更新state。

    • 负责修改state状态值
    • 数据源需要action提供
    function todoApp(state = initialState, action) {
  switch (action.type) {
    case SET_VISIBILITY_FILTER:
      return Object.assign({}, state, {
        visibilityFilter: action.filter
      })
    default:
      return state
  }
}

    # 合并多个reducer

    可以使用combineReducers函数来合并多个reducer

    combineReducers 辅助函数的作用是，把一个由多个不同 reducer 函数作为 value 的 object，合并成一个最终的 reducer 函数，然后就可以对这个 reducer 调用 createStore 方法。

    合并后的 reducer 可以调用各个子 reducer，并把它们返回的结果合并成一个 state 对象。 由 combineReducers() 返回的 state 对象，会将传入的每个 reducer 返回的 state 按其传递给 combineReducers() 时对应的 key 进行命名。

    // reducers/todos.js
export default function todos(state = [], action) {
  switch (action.type) {
    case 'ADD_TODO':
      return state.concat([action.text])
    default:
      return state
  }
}

// reducers/counter.js
export default function counter(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1
    case 'DECREMENT':
      return state - 1
    default:
      return state
  }
}

// reducers/index.js
import { combineReducers } from 'redux'
import todos from './todos'
import counter from './counter'

export default combineReducers({
  todos,
  counter
})

// App.js
import { createStore } from 'redux'
import reducer from './reducers/index'

let store = createStore(reducer)
console.log(store.getState())
// {
//   counter: 0,
//   todos: []
// }

store.dispatch({
  type: 'ADD_TODO',
  text: 'Use Redux'
})
console.log(store.getState())
// {
//   counter: 0,
//   todos: [ 'Use Redux' ]

    # store（数据管理的仓库）

    是redux的中心管理者，管理着redux中的状态，也提供相关方法来连接reducer和action。

    • 提供 getState() 方法获取 state；
    • 提供 dispatch(action) 方法更新 state；
    • 通过 subscribe(listener) 注册监听器;
    • 通过 subscribe(listener) 返回的函数注销监听器。

    # 创建store

    import { createStore } from 'redux'
// 假设已经有一个写好的reducer
import todoApp from './reducers'
let store = createStore(todoApp)

    # 使用redux的思路

    1. 分析业务逻辑

      1. 要托管的数据
      2. 怎么修改数据(方向)

    2. 使用redux

      1. 创建store对象

      2. 创建reducer

        1. 关注的是他的行为
        2. reducer：返回状态值给store对象
        3. 初始化给store对象状态值
        4. 分发action的时候，根据action的type类型以及action.payload决定如何修改状态，并将修改后状态值交给store对象
        5. 如何去匹配action.type---->switch case

      3. 设计action

        1. 本质：{type:行为，payload:数据}
        2. 定义函数：为了能够动态的设置payload数据

      4. 组件和redux交互

        1. 组件中一定要获取到store对象
        2. 获取redux中的状态：store.getState()
        3. 组件中分发action：store.dispatch(action对象)

    # react-redux

    # 理解

    1. 一个react插件库
    2. 专门用来简化react应用中使用redux

    # react-redux将所有组件分成两大类

    UI(展示)组件 容器组件
    作用 描述如何展现（骨架、样式） 描述如何运行（数据获取、状态更新）
    直接使用 Redux 否 是
    数据来源 props 监听 Redux state
    数据修改 从 props 调用回调函数 向 Redux 派发 actions
    调用方式 手动 通常由 React Redux 生成

    总结：

    1. UI(展示)组件

      1. 只负责UI的呈现, 不带有任何业务逻辑, 几乎没有和redux偶合
      2. 通过props接收数据(一般数据和函数)
      3. 不使用任何redux的api
      4. 一般保存在components文件夹下

    2. 容器组件

      1. 负责管理数据和业务逻辑, 不负责UI的呈现
      2. 使用redux的api
      3. 一般保存在containers文件夹下

    # mapStateToProps?: (state, ownProps?) => Object

    一个mapStateToProps函数最多接受两个参数。声明的函数参数（也称为arity）的数量会影响何时调用它。这也确定该函数是否将接收ownProps。

    1. 一个参数
    // 如果mapStateToProps函数声明为采用一个参数，
// 则只要存储状态发生更改，就会调用该函数，并将存储状态作为唯一参数。
const mapStateToProps = state => ({ todos: state.todos })
    1
    2
    3
    1. 两个参数
    // 如果mapStateToProps函数声明为带有两个参数，
// 则只要存储状态更改或**包装器组件收到新的props（基于浅层相等性比较）**，
// 就会调用该函数。将把存储状态作为第一个参数，并将包装器组件的props作为第二个参数。
// 第二个参数通常按惯例被称为ownProps。
const mapStateToProps = (state, ownProps) => ({
  todo: state.todos[ownProps.id]
})
